Verba API Documentation

RESTful API endpoints for the Document Q&A Assistant. All endpoints require authentication.

Authentication: All API endpoints require a valid login session. Include the CSRF token in the X-CSRFToken header for POST/DELETE requests.

Chat Sessions

GET /api/sessions/ List all chat sessions

Returns up to 30 most recent sessions ordered by last activity.

{
  "sessions": [
    {
      "id": 1,
      "title": "My Chat",
      "created_at": "2026-01-01T00:00:00Z",
      "updated_at": "2026-01-01T00:00:00Z"
    }
  ]
}
POST /api/sessions/create/ Create a new chat session

Creates a new chat session for the authenticated user.

// Response
{
  "id": 5,
  "title": "New Chat",
  "created_at": "2026-01-01T00:00:00Z"
}
GET /api/sessions/{id}/ Get session details with messages & docs

Returns full session including all messages and documents.

Parameter Type Description
id integer Session ID required
DELETE /api/sessions/{id}/delete/ Delete a chat session

Permanently deletes a session and all its messages, documents, and Pinecone vectors.

GET /api/sessions/{id}/export/ Export chat as Markdown

Downloads the full conversation as a Markdown file.

GET /api/search/?q={query} Search across sessions

Full-text search across session titles and message content.

Parameter Type Description
q string Search query required

Documents

POST /api/upload/ Upload a document

Upload a PDF, DOCX, or TXT file (max 10MB). The document is automatically chunked, embedded, and indexed in Pinecone.

Parameter Type Description
file file Document file required
session_id integer Session to attach to optional

Content-Type: multipart/form-data

{
  "id": 3,
  "name": "report.pdf",
  "status": "ready",
  "error": ""
}
GET /api/documents/?session_id={id} List documents

List all documents, optionally filtered by session.

DELETE /api/documents/{id}/delete/ Delete a document

Deletes the document, its file, and all associated Pinecone vectors.

Questions & Answers

POST /api/ask/ Ask a question (standard)

Submit a question and receive a complete answer with source citations.

Parameter Type Description
question string Your question (max 2000 chars) required
session_id integer Session context optional
model string LLM model ID optional 
// Request body (JSON)
{
  "question": "What is the revenue?",
  "session_id": 1,
  "model": "llama-3.1-8b-instant"
}

// Response
{
  "answer": "Based on the document...",
  "sources": [
    {
      "document_id": "3",
      "document_name": "report.pdf",
      "chunk_index": 2,
      "content": "..."
    }
  ]
}
POST /api/ask/stream/ Ask a question (streaming SSE)

Streaming version — tokens arrive via Server-Sent Events as the LLM generates them.

Response: text/event-stream

// Each SSE event:
data: {"type": "token", "content": "Based"}
data: {"type": "token", "content": " on"}
data: {"type": "token", "content": " the"}
...
data: {"type": "done", "answer": "Full answer...", "sources": [...]}

Configuration

GET /api/models/ List available LLM models

Returns all available models and the current default.

{
  "models": [
    {"id": "llama-3.1-8b-instant", "name": "Llama 3.1 8B", "provider": "groq", "speed": "fast"},
    {"id": "mixtral-8x7b-32768", "name": "Mixtral 8x7B", "provider": "groq", "speed": "medium"}
  ],
  "default": "llama-3.1-8b-instant"
}
